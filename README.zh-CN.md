# NGUInjector 中文版

NGUInjector 是面向 Steam 版 **NGU IDLE** 的自动化注入器。本分支在原项目基础上加入了中文界面、中文提示文本和更适合 NGU IDLE 汉化版术语的说明文档。

> 原项目：<https://github.com/rus9384/NGUInjector>

如果想了解各功能的触发条件、优先级、换装恢复和常见卡住原因，请阅读 [中文使用逻辑说明](USAGE.zh-CN.md)。

## 下载安装

请下载发布包中的 `NGUInjector-Chinese-compatible-v4.1.7-cn.4.zip`，不要下载 GitHub 自动生成的 Source code 压缩包。解压后目录中会有一个 `dist` 文件夹，主要文件如下：

```text
dist/
  inject.bat
  injector/
    NGUInjector.dll
    SharpMonoInjector.dll
    smi.exe
  sampleprofiles/
```

使用方法：

1. 在 Windows 上启动 Steam 版 NGU IDLE，并进入游戏。
2. 解压发布包到任意英文路径目录。
3. 运行 `dist/inject.bat`。
4. 注入成功后，游戏左上角会出现中文悬浮层，例如 `自动化 - 运行中/未运行`。
5. 按 `F1` 打开中文设置界面。

升级新版本时，请先在设置界面的 **常规** 页勾选确认框并点击 **卸载**，然后再运行新版本的 `inject.bat`。

## 配置目录

首次注入后，会自动创建配置目录：

```text
%UserProfile%\AppData\LocalLow\NGUInjector
```

常用文件：

- `settings.json`：注入器设置。大部分选项都可以在 `F1` 设置界面中修改。
- `zoneOverrides.json`：覆盖默认区域属性，用于黄金狙击计算。
- `profiles/default.json`：默认分配配置，控制装备、能量、魔力、资源3、Wandoos、重生、胡子和掘金者等自动化。

日志目录：

- `logs/inject.log`：注入器通用日志。
- `logs/loot.log`：敌人掉落日志。
- `logs/combat.log`：战斗算法日志。
- `logs/pitspin.log`：钱坑、每日转盘、世界树收获等结果。
- `logs/cards.log`：卡牌打出和丢弃日志。
- `logs/debug.log`：错误日志。设置界面打不开或配置不生效时先看这里。

保存 `settings.json`、`zoneOverrides.json` 或任意配置文件后，注入器会自动重新读取配置，不需要重启游戏。

## 设置界面

游戏内按 `F1` 打开设置界面。

### 常规

- **总开关**：关闭后禁用全部自动化。
- **禁用悬浮层**：隐藏游戏左上角状态显示。
- **钱坑运行模式**：自动向钱坑投入黄金，并通过重生最大化钱坑收益。
- **自动挑战首领**：自动核爆/挑战首领。
- **自动购买能量/魔力/资源3**：按自定义上限自动用经验购买属性。
- **自动购买冒险属性**：按自定义上限自动购买冒险属性。
- **自动购买消耗品**：自动购买配置文件中需要但当前没有的消耗品，仅在 AP 足够时购买。
- **已有加成时仍使用消耗品**：控制消耗品效果还在时是否继续使用。
- **自动每日存档**：AP 奖励可用时自动创建每日存档。
- **打开设置文件夹**：打开配置、档案和日志目录。
- **卸载**：从游戏中卸载注入器。

### 分配

- **分配配置**：切换 `profiles` 文件夹中的配置文件。
- **管理能量/魔力/资源3**：按当前配置文件中的断点自动分配资源。
- **管理 Wandoos OS**：按断点自动分配 Wandoos。
- **管理 NGU 难度**：按配置切换普通/邪恶/变态 NGU。
- **管理胡子**：按断点启用胡子。
- **管理掘金者**：按断点启用掘金者。
- **自动升级掘金者**：有钱时自动升级最便宜的掘金者，并尊重钱坑阈值。
- **掘金者限制为 GPS 的 X%**：限制掘金者花费占总 GPS 的比例。
- **管理装备**：按断点换装。
- **管理消耗品**：按配置文件自动使用药水、幸运符、松饼等。
- **自动重生**：达到配置文件中的时间或条件后自动重生。
- **自动血魔法**：自动施放血魔法，包括意大利面、数字加成、伪造黄金、铁丸、古夫 A/B 等。

### 世界树

- **管理世界树**：自动收获水果。某些情况下会在非最高等级收获古夫 B 果或怪癖果。
- **切换掘金者**：收获前启用 PP 和 EXP 掘金者。
- **切换胡子**：收获前启用黄金胡子。
- **切换世界树装备配置**：收获前装备指定物品。
- **激活水果**：重新激活尚未购买经验特权的水果。
- **最低切换等级**：低于该等级的水果不切换装备。
- **立即全部收获**：确认后立即切换并收获全部水果，适合手动重生前使用。

### 物品栏

- **管理物品栏**：启用后自动合并、自动 Boost、自动过滤、自动转换可转换物、任务物品管理等。
- **管理 Boost 转换**：完成普通 100 级挑战后，根据目标物品计算需要的 Boost 并自动调整转换。
- **魔方 Boost 优先级**：可选无、平衡、软上限、力量、韧性。
- **偏好 MacGuffin**：施放古夫 B 或吃古夫 A 果时，只装备选中的 MacGuffin。
- **装备 Boost 优先级**：默认为 `力量 > 韧性 > 特殊`。
- **优先 Boost 物品**：优先强化的物品列表。
- **黑名单物品**：永不强化或合并的物品列表。黑名单优先级高于优先列表。
- **重置 Boost 平均值**：重置魔方/装备强化的平均进度统计。

### 泰坦

- **管理泰坦**：启用泰坦自动战斗。
- **切换掘金者**：战斗前启用掉落率、冒险、PP、EXP 掘金者。
- **切换胡子**：战斗前启用 Neckbeard、BEARd 和黄金胡子。
- **切换装备配置**：战斗前装备泰坦配置。
- **泰坦版本**：快速选择泰坦版本。
- **战斗模式**：
  - **待机**：留在区域中待机。
  - **狙击**：满血后开战，预施放加成并谨慎使用野兽模式。
  - **防御**：约 80% HP 开战。
  - **进攻**：约 60% HP 开战，积极进攻。
  - **自动击杀**：泰坦刷新前换装，但不主动前往泰坦区域。
- **野兽模式**：待机战斗中是否使用野兽模式。
- **定位 Walderp**：Walderp 躲藏时跳转到它所在菜单。

### 冒险

- **启用战斗**：前往目标区域并自动战斗。
- **战斗模式**：待机、狙击、防御、进攻、一击。
- **目标区域**：冒险自动战斗或狙击的目标区域。
- **仅首领**：只打首领，适合区域狙击。
- **允许降级区域**：目标区域未解锁时，使用当前已解锁的最高区域。
- **ITOPOD**：无视目标区域，优先在 ITOPOD 战斗。泰坦战斗优先级更高。
- **ITOPOD 优化模式**：禁用、默认、PP、经验/任意点。
- **自动推进**：当前最高层低于最优层时尝试向上推进，失败后自动关闭。
- **敌人黑名单**：在指定区域中排除不想打的敌人。

### 黄金

- **启用黄金管理**：关闭后禁用所有黄金换装逻辑。
- **重新狙击时间**：到达指定重生时间后重新启用黄金狙击。
- **CBlock 模式**：解锁新区域时自动黄金狙击，适合挑战块。
- **立即黄金狙击**：立刻装备黄金配置，并根据区域属性计算最佳区域进行首领狙击。
- **泰坦黄金狙击**：勾选需要黄金换装的泰坦。绿色表示该泰坦本轮已完成黄金狙击。
- **重置泰坦状态**：立即把泰坦黄金状态重置为未完成。

### 任务

- **自动任务**：自动提交任务。
- **允许大型任务**：任务银行中有任务时，接受大型任务并手动刷指定区域物品。
- **任务银行满时开始大型任务**：任务银行已满时接受大型任务，否则接受小型任务。
- **大型任务使用黄油**：尝试给大型任务使用黄油。
- **手动小型任务**：小型任务也手动刷。
- **小型任务使用黄油**：尝试给小型任务使用黄油。
- **跳过 50 件以上小型任务**：跳过需求数量超过 50 的小型任务，会考虑当前进度。
- **放弃小型任务**：进度低于阈值时放弃小型任务，改接大型任务。
- **任务装备配置**：手动刷任务时装备的物品。

### 许愿

- **管理许愿**：按设置自动向许愿投入资源。
- **限制为 X 个许愿**：最多同时投入的许愿数量。
- **能量/魔力/资源3 %**：投入许愿的资源百分比。
- **分配模式**：
  - **默认**：按许愿编号优先。
  - **最便宜**：优先完成消耗最少的许愿。
  - **最快**：优先完成最快可完成的许愿。
  - **平衡**：空槽位中优先放最便宜且未拉黑的许愿，其余槽位给无法速度封顶的便宜许愿。
- **优先许愿**：优先占用许愿槽的列表。
- **软优先级**：开启后按分配模式排序，平局时参考优先列表顺序。
- **许愿黑名单**：永不投入的许愿。黑名单优先于优先列表。

### 钱坑

- **自动每日转盘**：自动转每日转盘并写入日志。
- **自动钱坑**：黄金达到阈值后自动投入钱坑。
- **预测钱坑结果**：投入前读取下一次钱坑奖励，并据此换装/分配资源。
- **从日托取出物品**：预测到装备冲击波时，从日托取出指定物品以最大化收益。
- **钱坑阈值**：等待达到该黄金数量后再投入钱坑。
- **日托阈值**：进度达到该百分比的日托物品不会被取出。
- **冲击波物品**：钱坑装备冲击波前要装备的物品。
- **切换掘金者和胡子**：投入钱坑前按预测奖励切换胡子和掘金者。

### 卡牌

- **管理蛋黄酱**：优先生产能尽快打出第一张卡牌的蛋黄酱；没有卡牌时均衡生产。
- **自动打出卡牌**：打出未保护且未被丢弃规则命中的卡牌。
- **打出已保护卡牌**：允许打出受保护卡牌。
- **排序卡牌**：按排序顺序整理卡牌。
- **排序顺序**：支持稀有度、阶层、花费、已保护、变化、价值、普通价值、类型等条件；带 `-ASC` 的规则反向排序。
- **自动丢弃卡牌**：按表格规则丢弃卡牌。
- **丢弃已保护卡牌**：允许按规则丢弃已保护卡牌。
- **丢弃设置**：按卡牌类型配置最低稀有度和花费阈值。

### 烹饪

- **管理烹饪**：自动优化烹饪效率。
- **为烹饪切换装备配置**：吃饭前切换到烹饪装备。
- **装备配置**：吃饭前装备的物品列表。

## 分配配置

分配配置位于 `profiles` 文件夹。它通过“重生时间断点”控制装备、能量、魔力、资源3、Wandoos、胡子、掘金者和重生逻辑。示例配置在 `sampleprofiles` 文件夹。

断点中的 `Time` 表示当前重生经过时间，单位为秒，也可以写成对象：

```json
"Time": {
  "h": 1,
  "m": 30,
  "s": 20
}
```

优先级可以追加百分比上限，例如：

```text
CAPRIT-0:30
```

表示尝试封顶第 0 个血仪式，但最多只投入当前上限的 30%。对非封顶优先级，则表示限制为闲置资源的百分比。

### 能量

```json
"Energy": [
  {
    "Time": 0,
    "Priorities": ["CAPNGU-0", "CAPWAN", "AT-1", "NGU-1"]
  }
]
```

封顶优先级会尽量用闲置能量达到未来 10 秒可 BB 的最高断点；非封顶优先级会按数量平分闲置能量并尝试达到断点。

常用能量优先级：

- `CAPNGU-X`、`CAPALLNGU`：封顶指定/全部能量 NGU。
- `CAPAT-X`、`CAPALLAT`：封顶高级训练。
- `CAPBESTAUG`、`CAPAUG-X`：封顶最佳/指定挂件。
- `CAPWAN`：封顶能量 Wandoos。
- `CAPTM`：封顶能量时间机器。
- `CAPBT-X`、`CAPALLBT`：封顶基础训练。
- `NGU-X`、`ALLNGU`：投入指定/全部 NGU。
- `AT-X`、`ALLAT`：投入高级训练。
- `AUG-X`、`BESTAUG`：投入挂件。
- `WAN`：投入 Wandoos。
- `TM`：投入时间机器。
- `BT-X`：投入基础训练。

### 魔力

```json
"Magic": [
  {
    "Time": 0,
    "Priorities": ["CAPNGU-0", "CAPWAN", "BR", "NGU-1"]
  }
]
```

常用魔力优先级：

- `CAPNGU-X`、`CAPALLNGU`：封顶指定/全部魔力 NGU。
- `CAPWAN`：封顶魔力 Wandoos。
- `CAPTM`：封顶魔力时间机器。
- `CAPRIT-X`：封顶指定血仪式。
- `BR`：从高到低施放可负担且 1 小时内完成的血仪式。
- `BR-X`：只施放能在指定秒数前完成的血仪式。
- `NGU-X`、`WAN`、`TM`、`RIT-X`：投入对应项目。

### 资源3

```json
"R3": [
  {
    "Time": 0,
    "Priorities": ["HACK-1"]
  }
]
```

- `(CAP)HACK-X`：投入指定黑客项目。
- `(CAP)ALLHACK`：加入全部黑客项目，并优先有目标值的项目。

### 装备、胡子、掘金者、Wandoos 和 NGU 难度

装备配置：

```json
"Gear": [
  {
    "Time": 0,
    "ID": [189, 442, 160, 441]
  }
]
```

胡子配置：

```json
"Beards": [
  {
    "Time": 0,
    "List": [5, 1, 6, 3]
  }
]
```

掘金者配置：

```json
"Diggers": [
  {
    "Time": 3650,
    "List": [8, 3, 4, 5]
  }
]
```

Wandoos：

```json
"Wandoos": [
  {
    "Time": 0,
    "OS": 1
  }
]
```

NGU 难度：

```json
"NGUDiff": [
  {
    "Time": 0,
    "Diff": 0
  }
]
```

这些列表大多从 0 开始编号。更多索引说明可参考原项目 wiki：<https://github.com/rus9384/NGUInjector/wiki/Allocation-Indexes>

### 重生

简单重生时间：

```json
"RebirthTime": -1
```

`-1` 表示不自动重生；其他数值表示到达指定秒数后重生。

复杂重生断点：

```json
"Rebirth": [
  {
    "Type": "Time",
    "Time": { "h": 1, "m": 1 }
  },
  {
    "Type": "Number",
    "Time": { "m": 15 },
    "Target": 10000
  }
]
```

可用类型：

- `Time`：到达指定时间后重生。
- `Number`：当前 Number 相比旧 Number 达到目标倍率后重生。
- `Bosses`：可额外击败指定数量首领后重生。
- `Muffin`：围绕 MacGuffin Muffin 优化 24 小时重生循环。
- `TimeBalancedMuffin`：类似 `Muffin`，但会平衡每日重生时间。

### 挑战

可在重生配置中指定进入挑战：

- `BASIC`：基础挑战
- `NOAUG`：无挂件挑战
- `24HR`：24 小时挑战
- `100LC`：100 级挑战
- `NOEC`：无装备挑战
- `TC`：巨魔挑战
- `NORB`：无重生挑战
- `LSC`：激光剑挑战
- `BLIND`：盲目挑战
- `NONGU`：无 NGU 挑战
- `NOTM`：无时间机器挑战

示例：

```json
"Challenges": ["BASIC-1", "BASIC-2", "TC-1", "NOEQ-1"]
```

### 消耗品

```json
"Consumables": [
  {
    "Time": 0,
    "Items": ["EPOT-B", "MPOT-B"]
  }
]
```

常用编号：

- `EPOT-A/B/C`：能量药水 Alpha/Beta/Delta
- `MPOT-A/B/C`：魔力药水 Alpha/Beta/Delta
- `R3POT-A/B/C`：资源3药水 Alpha/Beta/Delta
- `EBARBAR`：能量棒棒
- `MBARBAR`：魔力棒棒
- `MUFFIN`：MacGuffin Muffin
- `LC`：幸运符
- `SLC`：超级幸运符
- `MAYO`：蛋黄酱灌注器

可追加数量，例如：

```text
EPOT-A:2
```

这表示使用两个能量药水 Alpha。Beta 药水和 MacGuffin Muffin 会忽略数量，只会使用一个。

注意：注入器无法跨会话记住消耗品是否被手动使用。重启游戏、重启注入器或重新加载配置后，当前断点可能再次执行。它会尽量根据预计持续时间补偿，但不可能完美判断所有手动使用、重叠断点和共享计时器情况。

## 其他快捷功能

- `F2`：全局启用/禁用自动化。
- `F3`：快速保存，并导出可给 Gear Optimizer 使用的 `ngusav.es` JSON。
- `F5`：把当前装备写入日志，方便复制进分配配置。
- `F7`：读取快速保存。
- 自动监控冒险日志，把掉落写入 `loot.log`。
- 自动记录钱坑、每日转盘和世界树结果到 `pitspin.log`。
- 锁定物品栏中的 Boost 后，自动合并同类 Boost 直到 100 级。

## 中文化与兼容性说明

本分支做了以下改动：

- 翻译 `F1` 设置界面中的标签、按钮、下拉项和错误提示。
- 翻译游戏悬浮层、快速保存/读取提示、区域名称、卡牌稀有度、MacGuffin 类型和锁定动作名称。
- 保留内部配置键名，例如 `Power`、`Toughness`、`Special`、`RARITY`、`TYPE:xxx` 等，避免破坏原配置兼容性。
- 将 WinForms 资源改为静态赋值，绕开 Unity/Mono 对新版 `.resources` 的兼容问题。
- 移除发布包中的多余 `System.*.dll` 依赖，发布目录只包含注入所需文件。

如果 `F1` 设置界面初始化失败，请确认使用的是最新中文兼容版，并解压到全新目录运行。仍有问题时查看：

```text
%UserProfile%\AppData\LocalLow\NGUInjector\logs\debug.log
```
